# Simulador de terminal

> **Objetivo:** practicar el flujo de comandos del curso en un shell seguro.
> No ejecutas nada real; el simulador responde con salidas preparadas que
> reflejan lo que verías al seguir los labs y módulos.

Elige un escenario, escribe `help` + Enter para listar los comandos
disponibles, o copia y pega uno de los listados. Funciona `clear`, `ls`,
`echo <texto>` y las flechas para editar la línea.

**Cómo usarlo:**

1. Selecciona un **escenario** (lab-01, lab-02, M6 verificación, M3 workbench).
2. Escribe `help` para ver los comandos de ese escenario.
3. Ejecuta un comando (por ejemplo `bash init.sh` o `npm test`) y observa la
   salida: reportes DONE/VERIFIED, resultados de tests, mutation testing.
4. Cambia de escenario para ver otro flujo.

El simulador no toca tu filesystem. Es un entorno de práctica: el valor está
en reconocer los comandos y leer las salidas con ojo crítico — exactamente lo
que harás al seguir un agente en una sesión real.
